# Support

Thank you for using KubeNexus Scheduler! This document explains how to get help with the project.

## How to Get Help

### Documentation

Before opening an issue, please check our documentation:

- **[README](README.md)** - Project overview and quick start
- **[Architecture](docs/architecture.md)** - System design and components
- **[Hybrid Scheduling Guide](docs/HYBRID_SCHEDULING.md)** - Hybrid scheduling features
- **[Operator CRD Support](docs/OPERATOR_CRD_SUPPORT.md)** - Integration with Kubeflow, Spark, Argo, and Ray
- **[Comparison & Roadmap](docs/COMPARISON_AND_ROADMAP.md)** - Comparison with other schedulers

### Community Support

For questions, discussions, and community support:

- **GitHub Discussions** - Ask questions, share ideas, and discuss features
- **GitHub Issues** - Report bugs or request features (see [Contributing Guide](CONTRIBUTING.md))

### Bug Reports

If you've found a bug:

1. Check if the issue already exists in [GitHub Issues](https://github.com/YOUR_USERNAME/kubenexus-scheduler/issues)
2. If not, create a new issue using the [Bug Report template](.github/ISSUE_TEMPLATE/bug_report.md)
3. Provide as much detail as possible:
   - KubeNexus Scheduler version
   - Kubernetes version
   - Deployment configuration
   - Steps to reproduce
   - Expected vs. actual behavior
   - Relevant logs or error messages

### Feature Requests

Have an idea for a new feature?

1. Check existing [feature requests](https://github.com/YOUR_USERNAME/kubenexus-scheduler/issues?q=is%3Aissue+is%3Aopen+label%3Aenhancement)
2. If it doesn't exist, create a new issue using the [Feature Request template](.github/ISSUE_TEMPLATE/feature_request.md)
3. Describe:
   - The problem you're trying to solve
   - Your proposed solution
   - Alternative solutions you've considered
   - Use cases and benefits

### Security Issues

**Do not report security vulnerabilities through public GitHub issues.**

See our [Security Policy](SECURITY.md) for information on responsibly disclosing security vulnerabilities.

## Commercial Support

Currently, KubeNexus Scheduler is a community-driven open-source project without commercial support offerings. However:

- We welcome contributions from organizations
- Enterprise features and support may be available in the future
- Contact the maintainers for partnership opportunities

## Contributing

We welcome contributions! See our [Contributing Guide](CONTRIBUTING.md) for details on:

- Code contributions
- Documentation improvements
- Testing and bug reporting
- Community participation

## Response Time

As an open-source project maintained by volunteers:

- **Bug Reports**: We aim to acknowledge within 48 hours
- **Feature Requests**: We review periodically during roadmap planning
- **Pull Requests**: We aim to provide initial feedback within 1 week
- **Security Issues**: See [Security Policy](SECURITY.md) for timelines

Response times may vary depending on maintainer availability and issue complexity.
